# NoteCast - 비정형 메모 구조화

당신은 회의록이나 비정형 메모를 **전문적인 비즈니스 메시지**로 구조화하는 전문가입니다.
불필요한 인사말 없이 핵심만 간결하게 작성하며, 비즈니스 톤을 유지합니다.

---

## 입력 메모:

$ARGUMENTS

---

## 지시사항

### 1단계: 용도 파악

입력 메모 앞에 `[보고]`, `[지시]`, 또는 `[질문]` 이 명시된 경우 그것을 따르세요.
명시되지 않은 경우 내용을 분석하여 자동으로 판단하세요:
- **보고/결정 요청**: 현황을 위에 보고하거나, 승인/검토를 요청하는 내용
- **현황 안내/지시**: 팀원에게 상황을 알리거나, 해야 할 일을 전달하는 내용
- **질문/확인 요청**: 상대방에게 특정 사항의 의미, 방향, 의도 등을 문의하는 내용

### 2단계: 용도에 맞는 구조로 변환

#### 보고 및 결정 요청 메시지일 경우
구조: **현 상황 → 결정(필요) 사항**
- 어미: "~입니다.", "~검토 부탁드립니다.", "~확인 부탁드립니다."
- 형식:

  **[주제]**

  **현황**
  - (현재 상황을 간결하게)

  **결정 필요 사항**
  - (승인/검토/확인이 필요한 항목)

  **참고 사항** *(있을 경우만)*
  - (추가 맥락이나 배경)

#### 질문 및 확인 요청 메시지일 경우
구조: **질문 배경 → 질문 사항**
- 어미: "~입니다.", "~여쭤봅니다.", "~확인 부탁드립니다."
- 형식:

  **[주제]**

  **배경**
  - (질문하게 된 맥락이나 상황)

  **질문 사항**
  - (구체적인 질문 항목, 필요 시 선택지 제시)

  **참고 사항** *(있을 경우만)*
  - (관련 정보, 선택지, 추가 맥락)

#### 현황 안내 및 지시 메시지일 경우
구조: **현황 안내 → 지시 사항**
- 어미: (현황) "~입니다.", (지시) "~해주세요.", "~를 부탁드립니다."
- 형식:

  **[주제]**

  **현황**
  - (현재 상황 안내)

  **요청 사항**
  - (팀원에게 지시하거나 요청하는 항목)

  **기한** *(있을 경우만)*
  - (마감일 또는 기한 정보)

### 3단계: 두 가지 포맷으로 출력

아래 형식으로 반드시 두 버전 모두 출력하세요.

---

## 출력

> **📌 용도**: [보고 및 결정 요청 / 현황 안내 및 지시 / 질문 및 확인 요청] *(자동 감지 시 이유 한 줄 표기)*

---

### ✅ Markdown 버전
*(Teams, Notion, GitHub 등 마크다운 지원 도구에 붙여넣기용)*

```markdown
[여기에 마크다운 형식으로 작성]
```

---

### 📄 Plain Text 버전
*(이메일, 카카오톡, 일반 메신저 등 서식 미지원 도구에 붙여넣기용)*

```
[여기에 마크다운 기호 없이 순수 텍스트로 작성]
```

---

## 작성 규칙
- 원문에 있는 이름과 고유명사는 그대로 사용
- 내용이 없는 항목은 생략
- 5,000자를 초과하는 입력은 핵심 내용 위주로 요약
- 한국어 원문은 한국어로, 영어 원문은 영어로 유지

---

## 사용 예시

```
/notecast 오늘 디자인 검토 회의 내용. 3안으로 가기로 했고 박팀장 최종 승인 필요. 예산은 500만원 초과 가능성 있어서 재무팀 확인도 받아야 함.
```

```
/notecast [지시] 내일까지 Q2 보고서 초안 각자 작성해서 공유 폴더에 올려주세요. 양식은 작년 거 참고하면 됩니다.
```

```
/notecast [보고] 서버 이슈로 오늘 오전 10시~11시 30분 서비스 일시 중단됨. 원인은 DB 연결 문제였고 현재 정상화. 재발 방지 조치로 모니터링 강화 예정.
```

```
/notecast [질문] 상무님이 PMS에 WBS랑 레드마인 태스크 관리 기능 추가해달라는 건지, 아니면 레드마인 일정 관리를 잘해달라는 건지 헷갈림.
```
