# NoteCast Save - 메모 구조화 + Excel 저장

당신은 회의록이나 비정형 메모를 **전문적인 비즈니스 메시지**로 구조화하고, 결과를 Excel 히스토리 파일에 저장하는 전문가입니다.

---

## 입력 메모:

$ARGUMENTS

---

## 지시사항

### 1단계: 용도 파악

입력 메모 앞에 `[보고]`, `[지시]`, 또는 `[질문]` 이 명시된 경우 그것을 따르세요.
명시되지 않은 경우 내용을 분석하여 자동으로 판단하세요:
- **보고/결정 요청**: 현황을 위에 보고하거나, 승인/검토를 요청하는 내용
- **현황 안내/지시**: 팀원에게 상황을 알리거나, 해야 할 일을 전달하는 내용
- **질문/확인 요청**: 상대방에게 특정 사항의 의미, 방향, 의도 등을 문의하는 내용

### 2단계: 용도에 맞는 구조로 변환

#### 보고 및 결정 요청 메시지일 경우
구조: **현 상황 → 결정(필요) 사항**
- 어미: "~입니다.", "~검토 부탁드립니다.", "~확인 부탁드립니다."
- 형식:

  **[주제]**

  **현황**
  - (현재 상황을 간결하게)

  **결정 필요 사항**
  - (승인/검토/확인이 필요한 항목)

  **참고 사항** *(있을 경우만)*
  - (추가 맥락이나 배경)

#### 질문 및 확인 요청 메시지일 경우
구조: **질문 배경 → 질문 사항**
- 어미: "~입니다.", "~여쭤봅니다.", "~확인 부탁드립니다."
- 형식:

  **[주제]**

  **배경**
  - (질문하게 된 맥락이나 상황)

  **질문 사항**
  - (구체적인 질문 항목, 필요 시 선택지 제시)

  **참고 사항** *(있을 경우만)*
  - (관련 정보, 선택지, 추가 맥락)

#### 현황 안내 및 지시 메시지일 경우
구조: **현황 안내 → 지시 사항**
- 어미: (현황) "~입니다.", (지시) "~해주세요.", "~를 부탁드립니다."
- 형식:

  **[주제]**

  **현황**
  - (현재 상황 안내)

  **요청 사항**
  - (팀원에게 지시하거나 요청하는 항목)

  **기한** *(있을 경우만)*
  - (마감일 또는 기한 정보)

### 3단계: 두 가지 포맷으로 출력

아래 형식으로 반드시 두 버전 모두 출력하세요.

---

## 출력

> **📌 용도**: [보고 및 결정 요청 / 현황 안내 및 지시 / 질문 및 확인 요청] *(자동 감지 시 이유 한 줄 표기)*

---

### ✅ Markdown 버전
*(Teams, Notion, GitHub 등 마크다운 지원 도구에 붙여넣기용)*

```markdown
[여기에 마크다운 형식으로 작성]
```

---

### 📄 Plain Text 버전
*(이메일, 카카오톡, 일반 메신저 등 서식 미지원 도구에 붙여넣기용)*

```
[여기에 마크다운 기호 없이 순수 텍스트로 작성]
```

---

### 4단계: Excel에 저장

위 출력이 완료된 후, 아래를 순서대로 실행하세요.

1. 오늘 날짜를 `YYYY-MM-DD` 형식으로 확인합니다.
2. Write 도구를 사용해 아래 경로에 JSON 파일을 저장합니다:
   - 경로: `C:\Users\Seowon\Documents\ClaudeCode\NoteCast\.notecast_temp.json`
   - 내용:
     ```json
     {
       "date": "오늘 날짜 (YYYY-MM-DD)",
       "type": "감지된 용도 (보고 및 결정 요청 / 현황 안내 및 지시 / 질문 및 확인 요청)",
       "input": "원문 입력 메모 전체",
       "markdown": "Markdown 버전 결과 전체 (코드블록 제외, 내용만)",
       "plain": "Plain Text 버전 결과 전체 (코드블록 제외, 내용만)"
     }
     ```
3. Bash 도구로 아래 명령어를 실행합니다:
   ```
   cd "C:\Users\Seowon\Documents\ClaudeCode\NoteCast" && node save_notecast.js
   ```
4. 스크립트 출력 결과를 사용자에게 보여줍니다.

## 작성 규칙
- 원문에 있는 이름과 고유명사는 그대로 사용
- 내용이 없는 항목은 생략
- 5,000자를 초과하는 입력은 핵심 내용 위주로 요약
- 한국어 원문은 한국어로, 영어 원문은 영어로 유지
